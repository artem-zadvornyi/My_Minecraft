# Справочник систем (текущее состояние)

Каждая система: где живёт, как работает, за что отвечает.

## Запуск приложения — main.py

1. На macOS: отключение шейдеров + подавление GL-ошибок (см. architecture.md).
2. `Ursina()` → `window.update_aspect_ratio()` → настройка окна, неба, тумана.
3. `World(WORLD_SEED)` → `find_spawn()` (первая сухая колонка от x=0) →
   `pregenerate()` — синхронная генерация 49 чанков (~0.2 с) до появления окна.
4. Создание `Player`, `Inventory`, `UI`, `CraftingUI`, `GameState`,
   `GameController`; `app.run()`.

Флаги: `--smoke` (самопроверка + скриншоты + выход), `--benchmark` (замер FPS + выход).

## Маршрутизация ввода

Ursina рассылает `input(key)` всем включённым сущностям. Распределение:

- **GameController.input** (main.py): G — режим; E/Esc — окно крафта; 1-9 и
  колесо — слот; ЛКМ-креатив — мгновенное разрушение; ПКМ — съесть/поставить.
  Мировые действия блокируются, пока открыт крафт или отпущена мышь.
- **Player.input**: только двойной пробел (переключение полёта в креативе).
- **CraftingUI.input**: клики по слотам через `mouse.hovered_entity`
  (работает только когда `enabled`).
- Непрерывные действия (`held_keys`): WASD/пробел/shift в `Player.update`,
  удержание ЛКМ для разрушения в `GameController.update`.

## Игрок и физика — player.py

- `Player(Entity)`: позиция — ноги; камера прикреплена на 1.62.
- Обзор: `mouse.velocity * MOUSE_SENSITIVITY`, наклон ±89°.
- Движение по осям раздельно: сдвиг по x → разрешение коллизий по x → z → y.
  `_collide_axis` перебирает блоки, пересекающие AABB (≤ 2×3×2 = 12 проверок
  на ось), и выталкивает по грани. `EPS=1e-4` против «прилипания».
- Гравитация 25 бл/с², прыжок 8 бл/с (~1.25 блока), потолок скорости падения.
- Вода (по блоку на уровне ног): медленное погружение, пробел — всплытие,
  сбрасывает счёт высоты падения.
- Урон от падения: `_peak_y` — высшая точка с момента отрыва; при
  приземлении `on_hard_land(peak - y)` → `GameState.fall_damage`.
- Полёт (креатив): двойной пробел; вертикаль пробел/shift; коллизии работают.
- `frozen=True` (открыт крафт) — update пропускается целиком (пауза физики).
- Провал под y=-30 → `respawn()` на точку спавна.

## Мир — world.py

- **Шум**: `_make_noise(seed)` выбирает `noise` (C) → `perlin-noise` →
  встроенный Перлин. 3 октавы. Высота считается на грубой сетке шага 4
  с билинейной интерполяцией (`height_at`, кэшируется).
- **Генерация чанка** (`_generate_chunk`): колонки y=0..h — камень/земля/
  трава либо песок (h ≤ SAND_LEVEL); низины заливаются водой до
  WATER_LEVEL; деревья — детерминированный ГПСЧ на колонку (шанс 2%),
  только в глубине чанка, чтобы крона не пересекала границу; в конце
  накладываются `edits`.
- **Жизненный цикл чанков** (`update_chunks`, каждый кадр): при смене чанка
  игрока — пересчёт очереди недостающих (сортировка по дистанции) и
  выгрузка дальних (радиус > RENDER_DISTANCE+1). Не более ОДНОЙ генерации
  за кадр; после генерации соседи помечаются dirty и перестраиваются по
  одному за кадр (скрытие граней на границах).
- **Меши** (`_build_chunk_mesh`): для каждого блока чанка — 6 граней; грань
  добавляется, если сосед прозрачен/отсутствует. Вода — отдельная сущность
  (грани только к воздуху, полупрозрачность через `TransparencyAttrib`).
  Цвета граней с фиксированным затенением по стороне (верх 1.0 … низ 0.45),
  кэш в `_face_colors`. Сущность пересоздаётся при каждой перестройке.
- **Правки** (`set_block`): пишутся в `edits`, применяются к `blocks`,
  перестраивают меш чанка и, для граничных блоков, соседних чанков.
- **Рейкаст** (`raycast`): Аманатидес—Ву по сетке; вода прозрачна для луча;
  возвращает `(блок, клетка_перед_ним)` для разрушения/установки.

## Реестры — game_data/

- **BLOCKS** (8 блоков, id 1-8; 0 — воздух): `BlockDef` с полями
  hardness/tool/min_tool_tier, флагами (`transparent`, `solid`, `liquid`,
  `collision`, `replaceable`, `blocks_skylight`), светом (задел),
  дропами (`Drop(item, count, chance)`), текстурами граней
  (`FaceTextures`: одна / верх-низ-бок / по граням) и запасными цветами.
- **ITEMS** (13 предметов): предметы-блоки создаются автоматически из
  BLOCKS (`placeable_block` связывает с блоком), плюс палка, яблоко
  (`food=4`) и деревянные инструменты (`tool_type`, `tool_tier=1`,
  `mining_multiplier=4`, `durability` — задел). Палитра креатива — поле
  `creative_palette` (порядок).
- **mining.py**: `break_time` (учёт типа И уровня инструмента),
  `can_harvest` (задел под min_tool_tier), `get_drops` (списки с шансами,
  сидируемый rng для тестов).
- `validate()` при импорте: битые ссылки дроп→предмет и
  предмет→блок — ошибка запуска, а не тихий баг.

## Атлас текстур — rendering/texture_atlas.py

Все тайлы 16×16 из `assets/textures/` склеиваются в один атлас (+ белый
тайл №0). UV поджаты на полтекселя — с фильтрацией «ближайший сосед»
протекания нет. Раскладка и UV — чистый код (тестируется без окна),
GPU-текстура создаётся лениво. Грань без текстуры рисуется белым тайлом
с запасным цветом вершин; если атлас вообще недоступен — мир рисуется
цветами, как до М1. Иконки предметов — `apply_item_icon` (тайл атласа или
цветной квад с буквой). Текстуры — оригинальные, процедурные
(`tools/make_textures.py`, пересоздание детерминировано).

## Инвентарь — inventory.py

`Slot` — предмет/количество/`infinite`. `Inventory` — 9 слотов, `selected`,
`add()` (докладывает в стопки, потом в пустые; возвращает не вместившееся),
`consume_selected()`, `set_creative_palette()` (8 бесконечных блоков),
`set_slots()` (восстановление выживания). `on_change` — колбэк для HUD.
В креативе `add()` — no-op (подбор отключён).

## Крафт — crafting.py

- Рецепты: шаблоны 3×3 после `normalize` (обрезка пустых строк/столбцов) —
  рецепт работает в любом углу сетки. 6 рецептов (доски, палки, кирка,
  топор + зеркальный, лопата).
- `CraftingUI(Entity, parent=camera.ui)`: сетка 3×3, слот результата,
  зеркало инвентаря, стопка на курсоре. ЛКМ — вся стопка, ПКМ — половина/
  по одному. Результат: клик забирает на курсор и списывает по 1
  ингредиенту из занятых клеток.
- `open()`/`close()`: мышь отпускается/захватывается; при закрытии сетка и
  курсор возвращаются в инвентарь (в креативе — отбрасываются).

## Режимы игры — game.py

`GameState(Entity)`: `mode`, `health`, `hunger`, таймеры.

- **G** → `toggle_mode`: в креатив — инвентарь выживания сохраняется в
  `_survival_slots`, включается палитра и `can_fly`; обратно — восстановление.
- **Выживание**: голод −1 за 12 с; при 0 — урон 1/2 с; при сытости ≥16 —
  регенерация 1/3 с; `damage()` → `die()` → возрождение с полными шкалами
  (инвентарь сохраняется) + красная вспышка.
- **Креатив**: `damage()` игнорируется, шкалы скрыты.

## HUD — ui.py

Прицел, hotbar 9 слотов (иконка-цвет, буква инструмента, количество, номер),
рамка выбранного, имя предмета, полоски ХП/ЕДА (только выживание), текст
режима, подсказки управления, полоса прогресса разрушения, экран смерти.
Обновление — по колбэку `inventory.on_change` и явным вызовам
(`set_bars`, `set_mode`, `set_break_progress`).

## Конфигурация — settings.py

Все константы: чанки, шум, туман, физика игрока, инвентарь, выживание.
Модуль без зависимостей — безопасно импортировать откуда угодно.

## Тесты — tests/, tools/

## Сохранения

**Отсутствуют.** Всё состояние (правки мира, инвентарь, здоровье) живёт в
памяти процесса. Точка подключения: `World.edits` + слоты инвентаря +
`GameState` (миллстоун 11).
