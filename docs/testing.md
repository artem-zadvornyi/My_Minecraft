# Тестирование

Все команды — из корня проекта. Интерпретатор — `.venv/bin/python`
(зависимости установлены в виртуальное окружение).

## Быстрый прогон всего

```bash
.venv/bin/python -m py_compile settings.py inventory.py crafting.py world.py worldgen.py player.py ui.py game.py main.py game_data/*.py rendering/*.py
.venv/bin/python tests/test_registry.py   # ~1 с, без окна
.venv/bin/python tests/test_logic.py      # ~1 с, без окна
.venv/bin/python tests/test_worldgen.py   # ~10 с, без окна
.venv/bin/python tests/test_physics.py    # ~5 с, без окна
.venv/bin/python main.py --smoke          # ~5 с, открывает окно
.venv/bin/python tools/biome_shots.py     # фототур по биомам (окно)
```

Успех: `REGISTRY OK`, `LOGIC OK`, `WORLDGEN OK`, `PHYSICS OK`, `SMOKE OK`.

`tests/test_worldgen.py` покрывает детерминизм генерации: одно зерно —
один мир, разные зёрна — разный рельеф, независимость от порядка
генерации чанков, согласованность границ, покрытие всех 11 биомов
классификатором, воспроизводимость деревьев (включая кроны через границы
чанков), безопасность спавна (не в воде, не в блоках, равнина/лес).

`tests/test_registry.py` покрывает реестры и атлас: уникальность
key/id, валидность дропов и связей предмет→блок, лимиты стопок, еду,
время добычи с учётом инструментов, совместимость рецептов с реестром,
раскладку атласа (все грани всех блоков имеют тайлы, полутексельное
поджатие, отсутствие пересечений, фолбэк для неизвестных имён).

## Уровни тестов

### 1. tests/test_logic.py — чистая логика (без Ursina-приложения)

Рецепты и `normalize` (все 6, зеркальный топор, смещения по сетке,
негативные случаи), инвентарь (стопки 64, переполнение, креативная палитра,
восстановление выживания, нестопкуемость инструментов), блоки (`get_drop`,
`break_time`), генерация (детерминизм высот, послойность, вода ≤ уровня,
отрицательные чанки, наложение `edits` при перегенерации), координаты
чанков и воксельный рейкаст (осевые/диагональные лучи, вода прозрачна,
отрицательные координаты).

Работает без окна: модули импортируются, но `Ursina()` не создаётся,
меши не строятся (генерация трогает только словари).

### 2. tests/test_physics.py — интеграция физики (без окна)

Создаёт `Ursina(window_type='none')` и настоящего `Player`, прокручивает
`player.update()` с фиксированным dt=1/60 (детерминизм):
падение с точным приземлением и колбэком урона, ходьба 600 кадров по
рельефу без проваливаний, прыжок, полёт, `intersects_block`, рейкаст вниз,
вода (плавучесть, всплытие, отсутствие урона при падении в воду).

Особенности headless: `mouse.locked` подменяется заглушкой (нет окна);
`held_keys` выставляются напрямую.

### 3. python main.py --smoke — смоук в реальном окне

Полный запуск игры; через 2 секунды: проверки рецептов, разрушение/
установка блока в мире, рейкаст, стопки инвентаря, переключение
режимов с сохранением инвентаря, программная сборка лопаты в окне крафта
с расходом ингредиентов. Скриншоты `smoke_craft.png` (открытое окно
крафта) и `smoke.png` (итоговый кадр) — смотреть глазами после изменений
рендера. Завершается `SMOKE OK` и выходом.

## Замеры производительности

```bash
.venv/bin/python tools/benchmark.py   # без окна: генерация/меши/правки
.venv/bin/python main.py --benchmark  # окно: FPS (2 с прогрев + 6 с замер)
```

Базовые значения (16.07.2026, M-серия Mac, Python 3.14.6, ursina 8.3.0,
panda3d 1.10.16, шум `noise` 1.2.2):

```
                    после М1 (атлас)     после М2 (биомы, 6 слоёв шума)
Чанков / блоков:    49 / ~120 600        49 / ~196 900
Генерация чанка:    1.1 / 1.7 мс         3.4 / 10.5 мс     (медиана/макс)
Меш чанка:          2.6 / 11.5 мс        4.3 / 13.0 мс
Вершины:            122 392 + 452        144 992 + 26 508 (океаны)
Правка блока:       4.9 / 7.1 мс         4.4 / 8.1 мс
FPS средний/5%:     60.0 / ~57           60.0 / ~55
Пик памяти:         (не мерили)          ~217 МБ
```

М2: генерация подорожала (6 слоёв шума + биомы + деревья + украшения),
но генерация и меш теперь выполняются в РАЗНЫХ кадрах — пиковый кадр при
беге не хуже М1. FPS в статике не изменился.

Сравнивайте с этими числами после изменений генерации/мешей: деградация
медианы меша выше ~8 мс или правки выше ~10 мс будет заметна в игре.

## Чего тесты НЕ покрывают (проверять руками)

- Реальный ввод мыши/клавиатуры (обзор, клики по слотам крафта).
- Визуал: цвета граней, туман, прозрачность воды, вёрстка HUD/крафта
  (частично закрывается скриншотами smoke).
- Длительная игра: рост памяти кэшей, марафонские путешествия.
- Поведение при изменении размера окна.

## Правила для новых тестов

- Чистая логика — в test_logic.py (без `Ursina()`), физика — в
  test_physics.py (headless + фиксированный dt), всё остальное — смоук.
- Тесты — исполняемые скрипты с `assert` (pytest не подключён, чтобы не
  тянуть зависимость; при росте набора — подключить в миллстоуне 12).
- Не полагаться на порядок `update()` сущностей Ursina.
- Генерация детерминирована зерном: фиксируйте seed в тестах.
