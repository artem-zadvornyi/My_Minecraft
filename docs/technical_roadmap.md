# Технический роадмап производства

Это **действующий** план (заменяет прототипный [v2-roadmap.md](v2-roadmap.md)).
Стек: Godot 4.4+ / C# (обоснование — [engine_decision.md](engine_decision.md)).
Дизайн-цели фаз — из [game_design_document.md](game_design_document.md).

## Принципы миграции

1. **Прототип неприкосновенен**: Python/Ursina-код остаётся в репозитории
   работоспособным (его тесты гоняются до конца фазы P2 как спецификация).
2. **Инкрементально**: каждая фаза заканчивается играбельным билдом.
3. **Паритет до новизны**: сначала догоняем прототип (P0–P2), потом
   строим то, чего в нём не было. Исключение — атмосфера (свет/туман/
   звук) внедряется с P0: ради неё и мигрируем.
4. **Данные переезжают, код переписывается**: реестры экспортируются из
   Python одноразовым скриптом (JSON) и импортируются в Godot Resources;
   алгоритмы портируются по тестам, не по строкам.
5. **Никаких Entity-на-блок**: чанковые меши, как в прототипе.

## Структура репозитория (с P0)

```
/                  — прототип Python/Ursina (замораживается после P2)
godot/             — production-проект Godot (C#)
godot/data/        — реестры (экспорт из game_data/)
tools/export_registries.py — Python→JSON мост (пишется в P0)
docs/              — общая документация обоих миров
```

## Фазы

### P0 — Фундамент и вертикаль атмосферы (первый играбельный билд)

Каркас Godot-проекта: C#, структура сцен, фиксированный тик симуляции
(15 Гц) отдельно от рендера; чанковый мешер C# (порт архитектуры
прототипа: словарь блоков → меш чанка, границы, cross-растения);
экспорт/импорт реестров блоков; FastNoiseLite-поле высот (один слой —
достаточно для острова); игрок: ходьба/прыжок/AABB-физика (порт правил
прототипа); **атмосфера с первого дня**: Forward+, объёмный туман,
ночь, ручной фонарь, позиционный звук шагов.
**Плейбл-гейт**: ночная прогулка по острову с фонарём в тумане.
**Технический гейт**: сборки Win/macOS/Linux из CI; мешер ≤ 3 мс/чанк
в фоновом потоке (иначе — триггеры запасных планов из ADR-001).
Тесты: детерминизм мешера, физика с фиксированным dt (порт test_physics).

### P1 — Паритет мира

Порт пайплайна worldgen: слои шума (FastNoiseLite), классификатор
биомов, реки, деревья через границы чанков, украшения, спавн — вся
архитектура прототипа, но с северным набором биомов из GDD (болота,
гривы, ельники, гряды, пустоши, побережье, море). Пещеры (3D-шум +
штольни — новое). Стриминг чанков с бюджетом кадра.
**Плейбл-гейт**: свободное исследование, все биомы, пещеры.
Тесты: порт test_worldgen (детерминизм, границы, порядок, спавн).

### P2 — Паритет выживания (гейт паритета прототипа)

Добыча/установка блоков (рейкаст DDA), инвентарь/стеки, крафт по
рецептам-страницам, верстаки, тепло/еда/усталость, смерть/возрождение,
день/ночь/погода (дождь, туман, метель), сохранения (мир = зерно +
правки + состояние; формат версионируется с первого дня).
**Гейт паритета**: всё, что умеет прототип, умеет и Godot-билд, плюс
свет/туман/звук/пещеры/сохранения. После гейта Python-прототип
замораживается (остаётся в репо, тесты снимаются с CI).
**Плейбл-гейт**: полная суточная петля выживания, 60 FPS на средней
машине.

### P3 — Ночь живая: Просед, Внимание, первые Свидетели

Симуляция Внимания (источники: свет/шум/правки; затухание); симуляция
Проседа (память областей, бюджет пересборки, журнал изменений, границы
реализации из GDD §6); межевые знаки (подавление, ремонт, слом);
феномен-движок v1 (звуковые и предметные феномены из таксономии GDD);
**Свидетели волны 0 — Столпник и Гаситель** (обязательный состав
вертикального среза); прототип аудио-окклюзии (рейкасты + lowpass);
**картография v1**: планшет, съёмка с высоких точек, привязка к знакам
(механика, не автокарта); «дневник смелости».
**Плейбл-гейт**: вертикальный срез хоррора — «страшно вечером в
наушниках» (состав по production_vision, веха 1: Столпник + Гаситель +
Просед-минимум).
Тесты: симуляция Проседа headless (детерминизм, П1-причинность:
каждый феномен имеет запись о причине).

### P4 — Свидетели, волна 1 и публичное демо

Слушень (следопыт эха; окклюзия доводится до продакшена);
поведенческий стек Свидетелей: восприятие (свет/звук/взгляд) →
намерение → феномены; доводка Столпника и Гасителя; **настройки
интенсивности хоррора** (публичное демо обязано их иметь);
**юридическая очистка названия** до публикации страницы Steam
(поиск по WIPO/USPTO/EUIPO/Роспатент + поиск по Steam) и расширенный
чек-лист оригинальности (функциональные клоны существ любых игр, не
только Minecraft). Прототип «неправильных мест» (порталы-швы) —
оценка стоимости, решение по DLC-статусу.
**Плейбл-гейт**: демо-кандидат (Next Fest): 2–3 часа, съёмка карты,
пещеры со Слушнем, осада Гасителя.

### P5 — Территория рассказывает

Брошенные места по грамматикам (лагеря, деревни, станции, Контора);
страницы тетрадей (система знаний/рецептов; пишутся сразу RU/EN);
сезоны с «оседанием» года (удлинение ночей); огород и охота на
обычную фауну (сезонная еда — контраст Свидетелям); Отголосок и
Перенос; экономика «верных» инструментов.
**Плейбл-гейт**: ранний доступ (15–20 часов).

### P6 — Глубина и давление

Межевик, Наносной; штормовые события; глубинные пещеры и «холодное
стекло»; строительный набор Службы (латунь, стекло, свет за стеклом);
осадные ночи высокого Внимания.
**Плейбл-гейт**: контентная бета — осадная ночь высокого Внимания
проходима обоими стилями (оборона базы / ночная вылазка); глубинный
поход за «холодным стеклом» окупается и возвращаем.

### P7 — Финал и полировка

Триангуляционная сеть и «сдача съёмки»; концовки; баланс полного
прохождения; оптимизация (профили слабых машин); локализация RU/EN
финальная; достижения.
**Гейт**: релиз-кандидат 1.0.

### P8 — После 1.0

Кооп 2–4 (авторитетная симуляция уже отделена от рендера с P0);
«неправильные места» как контентное дополнение; поддержка модов
(реестры-как-данные делают это дёшево); консоли через партнёра.

## Оценки трудозатрат (порядок величин)

Допущения: 1 фултайм-разработчик + интенсивная ИИ-ассистенция +
точечный аутсорс (саунд-дизайн, ключевой арт). Оценки — калибруются
после P2 по фактической скорости; это честная цена соло-масштаба,
сжимается вторым человеком (арт/контент) после демо.

| Фаза | Оценка | Накопительно |
|------|--------|--------------|
| P0   | 2–3 мес | 3 мес |
| P1   | 2–3 мес | 6 мес |
| P2 (паритет) | 3–4 мес | ~год |
| P3   | 3–4 мес | ~1 г. 2 мес |
| P4 (демо) | 3–4 мес | ~1.5 года |
| P5 (ранний доступ) | 6–9 мес | ~2–2.5 года |
| P6+P7 (1.0) | 9–12 мес | ~3–3.5 года |

## Сквозные практики (все фазы)

- **Тесты**: детерминизм генерации и симуляций — headless C#-тесты
  (те же классы проверок, что в прототипе); феномен-журнал проверяется
  на П1-причинность автотестом.
- **Перфоманс-бюджеты**: кадр 16.6 мс: рендер ≤ 8, симуляция ≤ 3,
  мешинг — только фоновые потоки; замеры в CI на эталонной сцене
  (наследник tools/benchmark.py).
- **Атмосферный контроль**: каждая фаза заканчивается «ночным смотром»
  — скриншоты/запись эталонных сцен ночи и тумана (наследник
  tools/biome_shots.py).
- **Оригинальность**: чек-лист из GDD §11 (включая запрет функциональных
  клонов существ любых игр) — часть ревью контента каждой фазы.
- **Локализация**: RU/EN — двуязычие с P0 (все строки через ключи с
  первого экрана); тексты тетрадей в P5 пишутся сразу на двух языках.

## Что из прототипа куда переезжает

| Прототип | Судьба |
|----------|--------|
| game_data/* (реестры) | экспорт в JSON → Godot Resources (P0) |
| worldgen.py (архитектура) | порт на C# по тестам (P1) |
| world.py (чанки/меши/DDA) | порт архитектуры мешера (P0) и рейкаста (P2) |
| player.py (AABB-физика) | порт правил с фикс. dt (P0) |
| inventory/crafting | порт логики; UI — заново в Godot (P2) |
| tests/* | эталон поведения для портов (P0–P2) |
| tools/make_textures.py | остаётся генератором временного арта |
| Ursina-специфика (ui.py, атлас) | не переносится |
