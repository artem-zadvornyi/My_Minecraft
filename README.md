# Прототип «МЕЖА» — воксельная песочница на Python (Ursina)

> **Статус: играбельный прототип.** Проект переходит в production-фазу
> (оригинальный воксельный survival horror на Godot 4 + C#). Прототип
> сохраняется работоспособным как дизайн-референс и спецификация
> поведения. См. [docs/game_design_document.md](docs/game_design_document.md),
> [docs/engine_decision.md](docs/engine_decision.md),
> [docs/production_vision.md](docs/production_vision.md),
> [docs/technical_roadmap.md](docs/technical_roadmap.md).

Прототип: процедурный мир из чанков — 11 биомов (океаны, реки, пляжи,
леса, тайга, пустыни, саванны, горы со снегом, болота) на шести слоях
шума Перлина, три вида деревьев, инвентарь, крафт и два режима игры.

## Запуск

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python main.py
```

Самопроверка (запускает игру на пару секунд и прогоняет тесты логики):

```bash
.venv/bin/python main.py --smoke
```

## Управление

| Клавиша            | Действие                                    |
|--------------------|---------------------------------------------|
| WASD               | движение                                     |
| Мышь               | обзор                                        |
| Пробел             | прыжок (в воде — плыть вверх)                |
| Двойной пробел     | полёт вкл/выкл (только творчество)           |
| Shift              | в полёте — вниз                              |
| ЛКМ                | сломать блок (в выживании — удерживать)      |
| ПКМ                | поставить блок / съесть яблоко               |
| 1-9, колесо мыши   | выбор слота панели                           |
| E                  | окно крафта 3×3                              |
| G                  | режим: Выживание ⇄ Творчество                |
| Esc                | закрыть крафт / отпустить мышь               |

## Режимы

- **Выживание** — ресурсы ограничены, блоки ломаются со временем
  (инструменты ускоряют добычу), есть здоровье и голод, падение с высоты
  наносит урон, голод убывает со временем. Яблоки падают из листвы.
- **Творчество** — бесконечная палитра блоков, мгновенное разрушение,
  полёт по двойному пробелу.

## Крафт (E)

- Бревно → 4 доски
- 2 доски (столбиком) → 4 палки
- 3 доски + 2 палки → деревянная кирка
- 3 доски + 2 палки (углом) → деревянный топор
- 1 доска + 2 палки (столбиком) → деревянная лопата

В окне крафта: ЛКМ — взять/положить всю стопку, ПКМ — половину/по одному.

## Структура кода

| Файл / папка   | Назначение                                            |
|----------------|-------------------------------------------------------|
| `main.py`      | точка входа, привязка ввода к миру                    |
| `settings.py`  | все настройки                                          |
| `game_data/`   | реестры блоков, предметов и биомов, правила добычи    |
| `rendering/`   | атлас текстур и иконки предметов                       |
| `assets/`      | процедурные текстуры (`tools/make_textures.py`)       |
| `worldgen.py`  | слои шума, биомы, реки, деревья, украшения, спавн     |
| `world.py`     | жизненный цикл чанков, меши, воксельный рейкаст       |
| `player.py`    | камера от первого лица, физика, коллизии, полёт       |
| `inventory.py` | данные инвентаря                                       |
| `crafting.py`  | рецепты и окно крафта                                  |
| `ui.py`        | панель слотов, полоски здоровья/голода, прицел        |
| `game.py`      | режимы игры, здоровье, голод, урон                    |
| `tests/`       | тесты реестров, логики и физики                        |
| `tools/`       | замеры производительности, генератор текстур          |
| `docs/`        | архитектура, справочник систем, тесты, план V2        |

Подробная документация: [docs/architecture.md](docs/architecture.md),
[docs/current-systems.md](docs/current-systems.md),
[docs/testing.md](docs/testing.md), [docs/v2-roadmap.md](docs/v2-roadmap.md).

Производительность: каждый чанк 16×16 — один общий меш (плюс отдельный для
воды), рисуются только видимые грани; коллизии и выбор блока — по словарю
блоков, без физического движка. Чанки подгружаются по одному за кадр.
